Purpose
This document captures the screen-by-screen re=

Entity lookup removed. The fl=

Confirmation screen added. Af=

Host-controlled path selection. Verivend passes the verification path (Individual / Joint / Entity) as a=

Two-phase associated party model. The authorized representative enters biographical data for each associ=
ated person. Each person then independently completes identity verification=
via a standalone, token-authenticated link served by Interro outside the h=
ost application.
Session durability and integration hoo=

ebhook notifications, and retrieval API access are woven into the relevant =
screens.
